# 前端知识学习系统 · 需求文档

> 版本 v1.1 · 2026-05-14

---

## 一、产品定位

一个**纯本地运行**的前端工程知识学习系统，帮助中高级前端开发者系统性地学习工程化、性能、架构、浏览器与网络等知识，并通过随堂作业、模块刷题和错题训练检验掌握程度。

系统的内容目标是：对齐腾讯、阿里、字节等国内大厂，以及 Google、Meta 等国际公司对中高级前端的知识深度要求，同时训练用户解决真实客户问题的能力。

---

## 二、核心功能模块

### 2.1 学习模块（Learn）

| 要素 | 说明 |
|------|------|
| 内容组织 | 按**模块 → 篇**两级组织，每个模块下有多篇连续性文档 |
| 文档风格 | 技术破冰负责导读；深度长文沿用其结构，并增加真实客户问题、大厂追问和参考资料 |
| 连续性 | 文档有前后顺序，前篇是后篇的铺垫（如：为什么需要CI → 如何搭建Pipeline） |
| 进度追踪 | 每篇可标记「已读」，侧边栏显示完成状态，模块卡片显示整体进度 |
| 导航 | 支持上一篇 / 下一篇导航，侧边栏可折叠展开各模块 |

### 2.2 随堂作业模块（Doc Quiz）

| 要素 | 说明 |
|------|------|
| 触发时机 | 读完一篇文档后，点击「已读 + 开始作业」或「做随堂作业」触发 |
| 题目来源 | 对应当篇文档的专属题目，考察该篇核心知识点 |
| 题目数量 | 每篇 5～10 题 |
| 答题体验 | 逐题作答，答后立即显示正确答案与解析 |
| 结果反馈 | 答题完成后展示得分、正确率、鼓励文案 |

#### 2.2.1 答案展示样式（需求变更 · 2026-05-14）

- 查看答案时，统一采用信息卡片样式展示，视觉结构需与基准截图一致：浅灰背景容器 + 分块文案。
- 每题答案信息按固定顺序展示：
  1. 正确答案（示例：`正确答案：B`，答案字母高亮）
  2. 官方解析（完整解释文本）
  3. 知识点（标签或文本列表）
  4. 关联题目（可点击跳转的题目链接）
- 文案标签（`正确答案`、`官方解析`、`知识点`、`关联题目`）需视觉上明显区分正文（如加粗）。
- 关联题目必须可点击，点击后进入对应题目或题目详情。
- 答案卡片在桌面端与移动端均需保持可读性，不允许出现标签换行错位、内容溢出遮挡。

### 2.3 模块刷题模块（Drill）

| 要素 | 说明 |
|------|------|
| 题目范围 | 整个模块所有篇的题目合并，**不局限于单篇** |
| 出题方式 | 每次随机打乱顺序，避免刷题顺序记忆 |
| 题目统计 | 显示模块总题数、已练题数 |
| 入口 | 顶部导航「刷题」→ 选择模块 → 开始 |

### 2.4 错题本模块（Wrong Book）

| 要素 | 说明 |
|------|------|
| 收集方式 | 随堂作业、模块刷题、错题练习提交后，答错题自动进入错题本 |
| 移除方式 | 错题再次答对后自动移出错题本 |
| 练习方式 | 支持全部错题练习，也支持按模块针对练习 |
| 复习闭环 | 答案解析中展示推荐阅读章节，可直接跳回对应文档复习 |

## 三、题目类型

| 类型 | 说明 |
|------|------|
| 单选题 | 4 个选项，选 1 个 |
| 多选题 | 4～6 个选项，选多个，需手动点「确认选择」 |
| 判断题 | 「对 / 错」两个选项 |

所有题目均有**解析**（答后展示），帮助理解原理，而非死记答案。

## 四、数据存储

- **存储引擎**：浏览器 IndexedDB（离线可用，无需服务端）
- **存储内容**：

| Store | 键 | 内容 |
|-------|----|------|
| `progress` | `moduleId__docIdx` | 文档已读状态 |
| `quizRecord` | autoIncrement | 每次随堂作业的得分记录 |
| `drillStat` | `qid` | 每道刷题题目的对/错统计 |
| `wrongBook` | `qid` | 错题记录、用户答案、正确答案、推荐阅读来源 |

- **内容数据**：主入口为 `data/learning-content.js`，当前只聚合两类内容：
  - `技术破冰/cards/*.md`：破冰导读，系统通过 `data/tech-breaker-content.js` 聚合并解析自测题
  - 语义化深度长文数据文件：直接写在 `data/*.js` 中，作为破冰之后的系统学习内容

| 技术破冰：性能优化 | 前端性能优化、性能指标与度量、LCP、INP、Long Task、CLS、资源缓存、渲染流程、性能预算、Chrome Memory |
| 技术破冰：前端进阶 | TypeScript 进阶、Monorepo 工程、微前端、组件库设计 |
| 技术破冰：交付与后端基础 | CI/CD、发布策略、DevOps、Jenkins、请求-响应全链路、REST、Node、SQL、Docker、K8s |

### 6.2 第一批深度长文

| 模块 | 篇数 | 目标 |

第二批承接技术破冰里“工程化主干待补”方向，补齐自动化测试、前端安全、SSR/RSC 和状态数据层。

## 八、用户旅程

```
首页（模块卡片 + 进度）
  └─ 点击模块卡片
       └─ 学习页（侧边栏 + 文档阅读）
            └─ 读完 → 标记已读 + 开始作业
                 └─ 随堂作业（逐题答题 → 得分结果）
                      └─ 继续下一篇 / 返回

顶部导航「刷题」
  └─ 选择模块
       └─ 随机刷题（整模块题库）→ 得分结果

顶部导航「错题本」
  └─ 查看错题分布
       └─ 全部错题 / 按模块错题练习
            └─ 答对自动移出错题本，答错保留并累加次数
                 └─ 查看推荐阅读章节 → 返回学习页复习
```

## 十、验收标准（新增）

### 10.1 答案展示样式验收

1. 任意题目点击查看答案后，页面出现统一答案卡片容器。
2. 卡片内必须完整出现四个分区：`正确答案`、`官方解析`、`知识点`、`关联题目`。
3. `正确答案`中的答案值需明显高亮，且与题目标准答案一致。
4. `关联题目`为可点击链接，点击行为正确可达。
5. 在常见分辨率下（桌面端与移动端）卡片内容无截断、重叠、明显错位。

## 十一、答题交互与题库详实化优化（需求变更 · 2026-05-22）

### 11.1 选项卡片样式优化
- 精简答题选项 `.quiz-option` 的 padding 为 `10px 14px`。
- 设置垂直居中对齐（`align-items: center`），确保单行选项在视觉上更加紧凑，同时使多行选项的序号圈与字能够良好排版。

### 11.2 隐藏答题界面的文章来源
- 在做题答题界面（提交答案前），完全不渲染表示该题目来源的文章标签。
- 仅在答题提交后，在解析中以“推荐阅读”形式作为参考链接展示，防止答题过程中来源标签对作答思路产生提示或干扰。

### 11.3 题干脱离文章背景自描述
- 对系统中所有核心模块及博客文档底部的 11 篇自测题干进行全局详实化重写。
- 在题干中显式补全技术语境与主语（如“在 Vue 3 中...”、“在前端性能优化中...”等），确保用户在错题本或刷题模式下，单看题目即可清晰理解题意，无需结合原本的出处文章。

## 十二、题目解析防拷打深度重构（需求变更 · 2026-05-23）

### 12.1 题目解析（explain）结构化与防拷打深度升级
- 针对系统中所有模块（Vue 核心与源码、性能诊断、工程化构建、架构与场景设计）的题目解析（explain/解析字段）进行全方位的重构与扩写。
- 升级后的解析必须严格以结构化的多行模板进行呈现，分为两个主要章节：
  1. **💡 它解决了什么问题**：精炼说明如果没有该底层机制，系统会在性能、内存泄漏、可维护性或工程规范上遭遇的致命痛点。
  2. **🔍 核心原理解析（防拷打）**：分点阐述底层数据结构、生命周期、时序流或核心 API 的运作细节，剖析其背后的工程设计与选型取舍，并向外延伸大厂面试追问和极端边界场景的应对之道。

### 12.2 前端排版适配
- 在 `src/index.css` 的 `.quiz-explain-text` 中添加 `white-space: pre-wrap;` 规则，确保解析字符串中的换行格式能够在页面正常排版呈现。

### 12.3 多 Agent 协同机制
- 采用多 Agent 分工模式，分别派生 `VueSpecialistAgent`、`PerformanceSpecialistAgent`、`EngineeringSpecialistAgent` 和 `ArchitectureSpecialistAgent` 独立进行核心文件的改写与优化，实现高效、高质量的内容并发交付。

### 12.4 重构任务全面补齐与调试归档 (第二阶段)
- **遗漏文件全覆盖**：全面补齐了第一阶段由于分包或路径遗漏的 5 篇 Vue 源码博客文档（包含 Diff 算法、Effect 触发与调度器、复杂表单验证、模板到渲染函数转换、异步组件原理共 60 道题目）以及 5 个 JS 语义化数据文件（`security-content.js`、`rendering-architecture-content.js`、`delivery-platform-content.js`、`testing-quality-content.js`、`state-data-content.js`），确保全系统所有题库的解析完成 100% 的深度重构。
- **语法故障修复**：解决了 Vue 源码 Markdown 处理脚本中由于跨行单引号语法冲突引起的 Node.js 执行异常。将字符串定义规范化为反引号，顺利运行并将全新解析注入各 Markdown 对应章节。
- **构建与交付保障**：通过 `npm run build` 静态打包验证，无任何编译或解析错误，全新防拷打多行文本解析在学习系统中流畅展示。
