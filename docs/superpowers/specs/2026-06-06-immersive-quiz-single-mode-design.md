# 沉浸模式下单题流极简做题系统设计规格

本规格说明书定义了“学习系统”中刷题与自测模块在“沉浸专注模式”（Zen Mode）下的单题级切换及极简提交规格。

## 背景与目标

为了提供极致无干扰的刷题体验，我们对现有的 5 题一页的做题逻辑进行了重构。在**组织结构上依然保留 5 题一页的批量提交模式**，但在**沉浸模式交互上一次只展示一题**。选择完后支持单选题和判断题的自动切题，或者通过手动控件/键盘方向键切换，并支持在有空题的情况下随时提交。

## 规格细则

### 1. 数据结构与状态统一
* 在 `quizState` 中新增 `activeQuestionIdx` 属性（默认值为 `0`），用于标识当前正在激活显示的题目全局索引。
* 修改 `jumpToQuestion(globalIdx)` 方法，在计算并更新 `currentPageIdx = Math.floor(globalIdx / pageSize)` 的同时，更新 `activeQuestionIdx = globalIdx`。
* 任何上一页/下一页翻页操作，均同步将 `activeQuestionIdx` 重置为对应新页面的首道题索引（即 `currentPageIdx * pageSize`）。

### 2. 沉浸模式单题流交互 (Zen Single-Question Flow)
* **单题按需渲染**：当 `immersiveMode` 为 `true` 时，刷题页面内仅渲染局部索引为 `activeLocalIdx = activeQuestionIdx - currentPageStart` 对应的单一题目组件（`QuestionBlock`），完全隐藏当前页内的其他 4 道题目。
* **自动切题**：
  * 当题目类型为单选（`single`）或判断（`judgment`）时，用户点击选项后，系统在 `300ms` 延迟（留足选中态视觉反馈时间）后自动调用切题方法，跳至下一题（`activeQuestionIdx + 1`）。
  * 当题目类型为多选（`multiple`）时，系统不自动跳题，以便用户可以继续选择其他选项。
* **极简手动切换控件**：在题目组件下方渲染一行轻量级导航：
  * **上一题按钮**：左箭头 `←` SVG 图标，若当前为该页第一题则置灰且禁用。
  * **进度文本**：超小字号 `activeLocalIdx + 1 / pageSize`（如 `1 / 5`）。
  * **下一题按钮**：右箭头 `→` SVG 图标，若当前为该页最后一题则置灰且禁用。
* **键盘快捷切题**：键盘的左、右方向键（`ArrowLeft` / `ArrowRight`）可在 5 题中前后快速移动单题焦点。

### 3. 提交与空题规则
* **取消全部答完限制**：移除原有的提交限制。无论当前页题目是否全部选择，提交按钮始终处于激活状态，随时允许提交（即允许包含未答的空题直接提交，空题被判为回答错误并展示解析）。
* **极简圆角细线按钮**：
  * 在沉浸模式下，将“确认提交”、“下一题”、“查看结果”等关键按钮，从粗重的实色背景方块，重构为透明底色、极细边框的圆角药丸型按钮（`border rounded-full`），悬停时淡入微光。
  * 移除未做完题时的灰色虚线“还需回答 X 题”提示块，只在未提交且至少完成切题流程时常驻显示极简的“确认提交”按钮。
* **零底色题目解析**：
  * 在沉浸模式下，提交后的题目解析盒（`explanation`）彻底剥离 `bg-primary-muted` 灰色底色。
  * 改用左侧极细淡色状态线（`border-l-2 pl-4`）进行包裹，使其文字优雅融入页面背景，减少大面积卡片色块的堆叠。

## 验证方案

### 自动构建验证
* 在根目录运行 `npm run build`，确保代码静态编译无误。

### 手动交互测试
1. 开启沉浸模式，检查是否每次仅展示一题。
2. 点击单选/判断选项，验证是否在 `300ms` 后平滑跳到下一题。
3. 点击多选项，验证是否不自动切题。
4. 使用键盘左右方向键和手动 `←`/`→` 控件切换题目，检查跳转是否顺畅。
5. 在有空题的情况下，在最后一题或任意题点击“确认提交”，检查是否能成功提交。
6. 检查提交后的解析盒子是否为无背景、带左边线的效果。
