# 免费数据库存储与云端同步方案

面向个人和分享朋友使用的前端学习系统，为解决多端数据同步、防数据丢失以及多用户数据隔离问题，特引入 Supabase 免费云端数据库作为存储服务，并与本地 IndexedDB 组成“离线优先、手动云同步”的混合架构。

---

## 一、 需求分析与目标

1. **多用户数据隔离**：
   - 系统支持多用户使用（主要为个人和分享给朋友）。
   - 每个用户需要有独立的账户体系（支持 GitHub 第三方快捷登录或邮箱登录）。
   - 每个用户只能读写自己的数据（学习进度、错题本、刷题记录等），不能跨用户越权访问。

2. **离线优先与高可用性**：
   - 默认主存储依旧为浏览器本地的 IndexedDB，保证在无网、断网或云端不可用时，系统依然能够流畅使用。
   - 所有刷题记录、宠物状态等高频更新依然直接写入本地。

3. **手动云端同步**：
   - 提供一个用户可见的“云同步”控制面板。
   - 用户可以手动点击“立即同步”按钮，触发本地 IndexedDB 与云端 Supabase 数据库的双向合并。
   - 支持显示上一次成功同步的时间。

4. **安全与防刷**：
   - 数据库接口需要通过 RLS (Row Level Security，行级安全策略) 保证即使 API 密钥暴露在前端，也无法被恶意篡改其他用户的数据。

---

## 二、 技术选型：Supabase

- **数据库**：托管的 PostgreSQL (免费额度 500MB 存储，足够个人和朋友轻量使用多年)。
- **认证**：内置 Supabase Auth (月活 5 万免费用户额度)，提供 GitHub 等 OAuth 登录。
- **客户端**：纯前端使用 `@supabase/supabase-js` 直连云端，无需开发与维护后端 API 服务。

---

## 三、 数据表结构设计 (Supabase)

为实现多用户数据隔离，每张表都必须包含 `user_id` (对应 `auth.users` 的 UUID) 字段，并开启行级安全策略 (RLS)。

### 1. `user_progress` (用户学习文章进度表)
- `id`: `text` (主键，格式为 `module_id__doc_idx`)
- `user_id`: `uuid` (外键，对应 `auth.users`，联合主键)
- `done`: `boolean`
- `updated_at`: `timestamp with time zone` (用于版本冲突判断)

### 2. `quiz_records` (测验历史记录表)
- `id`: `bigint` (自增主键)
- `user_id`: `uuid`
- `module_id`: `text`
- `doc_idx`: `integer` (可为空)
- `score`: `integer`
- `total`: `integer`
- `pct`: `integer`
- `time`: `timestamp with time zone`

### 3. `drill_stats` (刷题数据统计表)
- `qid`: `text` (主键，题目 ID)
- `user_id`: `uuid` (联合主键)
- `total`: `integer`
- `correct`: `integer`
- `wrong`: `integer`
- `last_correct`: `boolean`
- `updated_at`: `timestamp with time zone`

### 4. `wrong_book` (错题本表)
- `qid`: `text` (主键，题目 ID)
- `user_id`: `uuid` (联合主键)
- `details`: `jsonb` (存储错题的快照详情，包含 userAnswer, wrongCount, createdAt 等)
- `updated_at`: `timestamp with time zone`

### 5. `pet_state` (宠物状态表)
- `id`: `text` (主键，默认为 'pet_state')
- `user_id`: `uuid` (联合主键)
- `state`: `jsonb` (存储宠物的等级、XP、名字、修炼速度等)
- `updated_at`: `timestamp with time zone`

### 6. `pet_events` (宠物修炼事件记录表)
- `id`: `text` (主键)
- `user_id`: `uuid`
- `event`: `jsonb` (存储事件的详细 JSON 数据，如标题、时间、描述、类型等)
- `updated_at`: `timestamp with time zone`

## 四、 核心技术方案：手动双向同步算法

用户点击“立即同步”时，系统将执行以下双向合并策略：

### 1. 本地库升级与时间戳标记
在 IndexedDB 中，为所有相关表记录补充 `updatedAt` 时间戳字段。
- 每次用户本地刷题、阅读、宠物产生事件等写操作，都在写入本地 IndexedDB 时自动将 `updatedAt` 设为 `Date.now()`。

### 2. 双向合并逻辑
对于包含更新时间戳的每张表（`progress`、`drillStat`、`wrongBook`、`petState`、`petEvents`）：
1. **拉取数据**：同时获取本地全部数据与云端该用户的全部数据。
2. **双向比对**：
   - 设本地数据为 $L$，云端数据为 $C$。
   - **对于每一个唯一 ID**：
     - 若 $L$ 存在且 $C$ 不存在：将 $L$ 标记为“需要上传云端”。
     - 若 $C$ 存在且 $L$ 不存在：将 $C$ 标记为“需要写入本地”。
     - 若 $L$ 与 $C$ 均存在：
       - 对比 $L.updatedAt$ 与 $C.updatedAt$：
         - 若 $L.updatedAt > C.updatedAt$：说明本地数据更新，将 $L$ 标记为“需要上传云端”。
         - 若 $L.updatedAt < C.updatedAt$：说明云端数据更新，将 $C$ 标记为“需要写入本地”。
         - 若相等，则不处理。
3. **批量写回**：
   - 将“需要上传云端”的列表批量 Upsert 写入 Supabase。
   - 将“需要写入本地”的列表批量写入 IndexedDB。
   - 同步完成后，重新刷新前端的 React State 内存缓存，使用户即刻看到同步后的效果。

## 五、 UI 交互设计

1. **导航栏同步按钮**：
   - 在主界面右上角添加一个 `☁️ 云同步` 或 🔄 图标。
   - 当用户未登录时，显示置灰或警告标志。
   - 当用户已登录时，悬停显示“已登录，点击管理同步”。

2. **账户与同步管理弹窗 (Modal)**：
   - 采用精致的 Glassmorphism (毛玻璃) 风格设计。
   - **登录模块**：未登录用户可选择 GitHub 登录（推荐，可一键完成 OAuth 授权并跳转回本站）。
   - **同步控制模块**：
     - 显示当前账号。
     - 显示“上次同步时间：XXXX-XX-XX XX:XX”。
     - 提供“立即同步”按钮，伴随 Loading 微动画。
     - 提供“退出登录”按钮（退出时只清除内存登录状态，本地 IndexedDB 不会被抹去，防止误删用户数据）。

## 六、 安全规则 (RLS 示例)

每张表在 Supabase 中必须执行类似如下的 RLS 策略：

```sql
-- 启用 RLS
ALTER TABLE public.wrong_book ENABLE ROW LEVEL SECURITY;

-- 创建策略：仅允许用户操作属于自己的数据
CREATE POLICY "Users can only access their own wrong_book records"
ON public.wrong_book
FOR ALL
USING (auth.uid() = user_id)
WITH CHECK (auth.uid() = user_id);
```
