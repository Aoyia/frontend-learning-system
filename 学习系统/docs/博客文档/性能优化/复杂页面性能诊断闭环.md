# 复杂页面性能诊断闭环：从用户慢到工程治理

性能优化的中高级分水岭在于：你能不能把“页面慢”拆成可测量、可定位、可验证、可防回退的链路。

不要一上来背“懒加载、压缩、缓存”。面试官更想听到：

> 我会先把用户现象映射到指标，再用真实用户数据和本地录制定位瓶颈，最后用性能预算、监控告警和发布门禁防止回退。

## 1. 它解决什么问题？

复杂页面性能问题通常混在一起：

- 首屏慢：可能是 TTFB、资源发现晚、图片大、JS 阻塞、CSR 等。
- 点击慢：可能是主线程 Long Task、同步计算、渲染范围大。
- 页面跳：可能是图片、广告、异步组件、字体导致 CLS。
- 用久变慢：可能是内存泄漏、监听器未释放、缓存无界增长。
- 某地区慢：可能是 CDN、DNS、网络、后端节点。

中高级回答要先拆问题，而不是堆方案。

## 2. 性能诊断总流程

```txt
明确用户现象
  ↓
选择指标：LCP / INP / CLS / TTFB / Long Task / Memory
  ↓
看真实用户数据：RUM / CrUX / release 分组
  ↓
本地复现：Performance / Network / Memory
  ↓
定位瓶颈：网络、资源、渲染、JS、内存、后端
  ↓
设计方案：收益、成本、风险、回滚
  ↓
灰度验证：指标对比
  ↓
性能预算和告警防回退
```

## 3. LCP：首屏最大内容怎么变快？

LCP 不只是“图片压缩”。

如果 LCP 元素是首屏大图，应拆成：

| 阶段 | 常见问题 | 优化方向 |
| --- | --- | --- |
| TTFB | 后端慢、SSR 慢、缓存缺失 | CDN、缓存、服务端性能 |
| 资源发现 | 图片在 JS 后才出现 | HTML 中提前发现、preload |
| 资源加载 | 图片大、格式旧、CDN 慢 | AVIF/WebP、尺寸裁剪、缓存 |
| 渲染延迟 | JS 阻塞主线程 | 减少阻塞 JS、拆包、延后非关键脚本 |

注意：首屏 LCP 图片通常不应设置 `loading="lazy"`，否则可能推迟加载。

## 4. INP：点击卡顿怎么定位？

INP 关注用户交互到下一次绘制之间的响应延迟。

常见根因：

- 点击回调里同步处理大数组。
- 输入时触发大范围状态更新。
- 第三方 SDK 或埋点同步阻塞。
- 图表、富文本、表格初始化占满主线程。
- 长任务没有拆分或让出主线程。

定位步骤：

1. Performance 面板录制一次慢交互。
2. 找到 Interaction 和 Long Task。
3. 展开调用栈，看是业务计算、框架渲染还是第三方库。
4. 对照 React/Vue Profiler 看组件渲染范围。
5. 拆任务、移 Worker、虚拟列表、服务端分页或降低更新优先级。

## 5. CLS：页面为什么突然跳？

CLS 是视觉稳定性问题，不是加载速度问题。

高频原因：

- 图片、视频没有宽高或 `aspect-ratio`。
- 广告位、推荐位、异步组件没有预留空间。
- 字体加载导致文本尺寸变化。
- 错误提示插入在已有内容上方。
- 骨架屏和真实内容尺寸不一致。

面试里要强调：优化 CLS 的关键是**预留空间和稳定布局**，不是单纯加 loading。

## 6. 内存与长时间使用性能

客户说“页面打开半小时后越来越卡”，要优先怀疑：

- 事件监听器、定时器未释放。
- WebSocket / SSE 重连重复注册。
- 大数组缓存无上限。
- 图表、地图、编辑器实例未销毁。
- Detached DOM 被闭包引用。

排查工具：

- Chrome Memory heap snapshot。
- Performance Monitor 看 JS heap、DOM nodes。
- Allocation instrumentation 观察持续增长。
- 手动执行进入页面、离开页面、重复操作，看对象是否释放。

## 7. 性能预算与防回退

一次优化上线后，如果没有预算和监控，很容易下个版本退化。

可以建立：

- JS 首屏包体预算。
- LCP / INP / CLS P75 预算。
- 关键接口耗时预算。
- 图片体积和格式规则。
- Lighthouse CI 或自定义构建门禁。
- RUM 告警与 release 对比。

性能优化的终点不是“这次变快”，而是“以后不轻易变慢”。

## 10. 作业

1. 找一个复杂页面，录制首屏加载并标出 LCP 元素。
2. 把 LCP 拆成 TTFB、资源发现、资源加载、渲染延迟四段。
3. 制造一次点击卡顿，用 Performance 找出 Long Task 调用栈。
4. 制造一次布局偏移，使用 DevTools 找到 Layout Shift 来源。
5. 写一个页面反复进入退出的内存泄漏 demo，并用 Memory 面板证明对象没有释放。
6. 为项目写一份性能预算表，至少包含 JS 体积、LCP、INP、CLS。
