# 内容扩展规范

本题库目标是辅助用户成为能解决真实客户问题的中高级前端工程师，而不是只背概念。

## 内容定位

- 面向腾讯、阿里、字节等国内大厂中高级前端面试。
- 面向 Google、Meta 等国际公司对 Web 性能、架构、工程质量和问题定位能力的要求。
- 每篇文档都应该服务于一个真实工程问题，例如“客户反馈页面慢”“发布后部分用户白屏”“组件库升级导致业务项目构建失败”。

## 每篇文档结构

建议保持以下结构：

1. 真实问题或面试场景
2. 核心概念和链路
3. 排查或设计步骤
4. 面试版回答
5. 题目

## 题目要求

- 每篇至少 6 题。
- 题型覆盖单选、多选、判断。
- 题目应优先考察判断标准、取舍、定位路径，而不是死记名词。
- 每题必须有 `explain`，解释为什么正确，而不是只给答案。

## 数据组织

- `learning-content.js` 是主内容入口，`src/App.jsx` 只从这里读取学习内容。
- 技术破冰导读维护在 `../技术破冰/cards/*.md`，由 `tech-breaker-content.js` 聚合，并把文档末尾的“选择题自测”解析成系统内随堂作业。
- 深度学习长文直接写在语义化数据文件中，例如 `engineering-toolchain-content.js`、`performance-diagnostics-content.js`、`security-content.js`。
- 新增深度模块时，优先按知识领域命名文件，并在模块上记录 `sourceCards`，说明来自哪些技术破冰卡片。
- 旧的 `content.js`、`expanded-content*.js` 保留为历史数据，不进入主学习入口。
- 新模块应有稳定 `id`，不要随意改已有模块或题目的顺序；错题本依赖 `moduleId__docIdx__quizIdx` 作为题目定位。

## 推荐写作格式

深度长文优先采用 `技术破冰/cards` 的知识卡结构，并增加真实客户问题和大厂追问：

1. 它解决什么问题
2. 真实客户问题
3. 核心流程
4. 关键词清单
5. 一句面试版
6. 最小 demo / 最小案例
7. 常见问题怎么定位
8. 大厂面试追问
9. 是否值得深入
10. 参考资料
11. 选择题自测
