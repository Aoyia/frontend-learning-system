# 前端知识学习系统 · 需求文档

> 版本 v1.0 · 2026-05-12

---

## 一、产品定位

一个**纯本地运行**的前端工程知识学习系统，帮助中高级前端开发者系统性地学习 CI/CD、DevOps、Jenkins 等工程化知识，并通过随堂作业和模块刷题检验掌握程度。

---

## 二、核心功能模块

### 2.1 学习模块（Learn）

| 要素 | 说明 |
|------|------|
| 内容组织 | 按**模块 → 篇**两级组织，每个模块下有多篇连续性文档 |
| 文档风格 | 贴近真实开发场景，着重讲**解决了什么问题**，而非枯燥罗列概念 |
| 连续性 | 文档有前后顺序，前篇是后篇的铺垫（如：为什么需要CI → 如何搭建Pipeline） |
| 进度追踪 | 每篇可标记「已读」，侧边栏显示完成状态，模块卡片显示整体进度 |
| 导航 | 支持上一篇 / 下一篇导航，侧边栏可折叠展开各模块 |

### 2.2 随堂作业模块（Doc Quiz）

| 要素 | 说明 |
|------|------|
| 触发时机 | 读完一篇文档后，点击「已读 + 开始作业」或「做随堂作业」触发 |
| 题目来源 | 对应当篇文档的专属题目，考察该篇核心知识点 |
| 题目数量 | 每篇 5～10 题 |
| 答题体验 | 逐题作答，答后立即显示正确答案与解析 |
| 结果反馈 | 答题完成后展示得分、正确率、鼓励文案 |

### 2.3 模块刷题模块（Drill）

| 要素 | 说明 |
|------|------|
| 题目范围 | 整个模块所有篇的题目合并，**不局限于单篇** |
| 出题方式 | 每次随机打乱顺序，避免刷题顺序记忆 |
| 题目统计 | 显示模块总题数、已练题数 |
| 入口 | 顶部导航「刷题」→ 选择模块 → 开始 |

---

## 三、题目类型

| 类型 | 说明 |
|------|------|
| 单选题 | 4 个选项，选 1 个 |
| 多选题 | 4～6 个选项，选多个，需手动点「确认选择」 |
| 判断题 | 「对 / 错」两个选项 |

所有题目均有**解析**（答后展示），帮助理解原理，而非死记答案。

---

## 四、数据存储

- **存储引擎**：浏览器 IndexedDB（离线可用，无需服务端）
- **存储内容**：

| Store | 键 | 内容 |
|-------|----|------|
| `progress` | `moduleId__docIdx` | 文档已读状态 |
| `quizRecord` | autoIncrement | 每次随堂作业的得分记录 |
| `drillStat` | `qid` | 每道刷题题目的对/错统计 |

- **内容数据**：预生成，以 `data/content.js` 的形式硬编码，后续手动更新

---

## 五、技术栈

| 层 | 选型 |
|----|------|
| 框架 | 纯静态 HTML / Vanilla JS（无构建工具，直接打开即用） |
| Markdown 渲染 | [marked.js](https://marked.js.org/) CDN |
| 代码高亮 | [highlight.js](https://highlightjs.org/) CDN |
| 样式 | 手写 CSS，深色主题 |
| 存储 | IndexedDB（原生 API 封装） |

## 六、初始内容规划

### 模块一：CI/CD（6 篇）

| # | 篇名 | 核心问题 |
|---|------|----------|
| 1 | 手动部署之痛：为什么需要 CI/CD？ | 解决「每次上线都要人工操作、容易出错」的问题 |
| 2 | CI 是什么：代码提交后发生了什么？ | 解决「代码合并前如何自动保证质量」 |
| 3 | CD 是什么：从构建产物到线上环境 | 解决「构建产物如何安全、自动地部署到各环境」 |
| 4 | Pipeline 设计：stages、jobs 与并行 | 解决「流水线如何高效组织，减少等待时间」 |
| 5 | 环境管理：dev / staging / prod 如何隔离？ | 解决「多环境配置混乱、误发布到生产」 |
| 6 | 回滚策略：出了问题如何快速恢复？ | 解决「发布后发现 bug，如何最小化影响」 |

### 模块二：DevOps（5 篇）

| # | 篇名 | 核心问题 |
|---|------|----------|
| 1 | 开发与运维的墙：DevOps 从何而来？ | 解决「研发交付慢、上线故障频发」 |
| 2 | 基础设施即代码（IaC）：告别手动配置服务器 | 解决「服务器环境难以复现和版本管理」 |
| 3 | 容器化基础：Docker 解决了什么问题？ | 解决「环境不一致、依赖冲突」 |
| 4 | 监控与可观测性：系统出问题怎么第一时间知道？ | 解决「上线后无感知、排查问题靠猜」 |
| 5 | 前端 DevOps 实践：从代码到 CDN 的完整链路 | 解决「前端项目部署流程混乱」 |

### 模块三：Jenkins（5 篇）

| # | 篇名 | 核心问题 |
|---|------|----------|
| 1 | Jenkins 是什么：一台不知疲倦的自动化工人 | 解决「CI/CD 工具选型与 Jenkins 定位」 |
| 2 | 第一条 Pipeline：用 Jenkinsfile 描述流水线 | 解决「如何用代码定义构建流程（Pipeline as Code）」 |
| 3 | 插件生态：Jenkins 如何与各种工具集成？ | 解决「与 Git、Docker、SonarQube 等工具打通」 |
| 4 | 凭证管理：密钥和敏感信息怎么安全存储？ | 解决「Jenkinsfile 里不能硬编码密码的问题」 |
| 5 | 多分支与多环境：一个 Jenkinsfile 管理所有分支 | 解决「不同分支触发不同流水线逻辑」 |

## 七、页面结构

```
index.html          主入口（含全部 CSS + HTML 骨架）
app.js              应用逻辑（路由、渲染、IndexedDB、测验逻辑）
data/
  content.js        所有学习文档 + 题目数据（预生成 JS 对象）
PRD.md              本需求文档
```

---

## 八、用户旅程

```
首页（模块卡片 + 进度）
  └─ 点击模块卡片
       └─ 学习页（侧边栏 + 文档阅读）
            └─ 读完 → 标记已读 + 开始作业
                 └─ 随堂作业（逐题答题 → 得分结果）
                      └─ 继续下一篇 / 返回

顶部导航「刷题」
  └─ 选择模块
       └─ 随机刷题（整模块题库）→ 得分结果
```
