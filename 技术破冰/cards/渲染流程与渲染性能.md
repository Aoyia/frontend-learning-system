# 渲染流程与渲染性能

## 1. 它属于哪个知识板块？

```txt
前端性能优化
→ 浏览器渲染
→ HTML / CSSOM / layout / paint / composite
```

==渲染性能关注的是浏览器为了把 DOM 和 CSS 变成屏幕像素，到底做了多少工作。==

## 2. 陌生技术出现后的学习顺序

### 2.1 它解决什么问题？

==**渲染流程与渲染性能**解决的是“**代码执行完了，但浏览器布局、绘制、合成太重，页面还是卡**”的问题。==

很多卡顿不是 JS 算法慢，而是你让浏览器反复计算布局、反复重绘大面积区域、创建过多图层、滚动时不断读写 DOM 尺寸。

==优化渲染的核心是理解浏览器的**样式计算、布局、绘制、合成**链路，避免无意义的重复工作。==

### 2.2 核心流程

```txt
HTML 解析成 DOM
→ CSS 解析成 CSSOM
→ DOM + CSSOM 生成 render tree
→ layout 计算每个盒子的位置和尺寸
→ paint 把视觉样式绘制成像素
→ composite 把图层合成到屏幕
→ JS / CSS / DOM 变化触发部分流程重跑
```

### 2.3 关键词清单

1. DOM：页面结构树。
2. CSSOM：CSS 规则树。
3. render tree：真正参与渲染的节点树。
4. style recalculation：样式重新计算。
5. layout / reflow：计算元素几何位置和尺寸。
6. paint / repaint：绘制颜色、文字、边框、阴影等。
7. composite：合成图层。
8. layout thrashing：读布局属性和写样式交替，强制多次同步布局。
9. composited layer：独立合成图层。
10. transform / opacity：通常只影响合成，更适合动画。
11. containment：用 CSS containment 缩小布局和绘制影响范围。
12. virtualization：虚拟列表，避免一次渲染大量 DOM。

### 2.4 一句面试版

==浏览器渲染会经历 **DOM、CSSOM、layout、paint、composite** 等阶段；优化渲染性能的核心是减少大范围样式重算、布局和绘制，避免 layout thrashing，大列表做虚拟滚动，动画优先用 **transform / opacity**，必要时用 containment 缩小影响范围。==

### 2.5 最小 demo / 最小案例

容易触发布局抖动的写法：

```ts
for (const item of items) {
  const height = item.offsetHeight
  item.style.height = `${height + 10}px`
}
```

更好的思路：

```ts
const heights = items.map(item => item.offsetHeight)

items.forEach((item, index) => {
  item.style.height = `${heights[index] + 10}px`
})
```

动画优先：

```css
.panel {
  transform: translateX(0);
  transition: transform 200ms ease;
}

.panel.open {
  transform: translateX(100px);
}
```

少用会触发布局的动画：

```css
/* 尽量避免用 left/top/width/height 做高频动画 */
```

### 2.6 常见优化手段

1. 批量读 DOM，再批量写 DOM。
2. 大列表使用虚拟滚动。
3. 动画优先使用 transform / opacity。
4. 减少复杂 CSS 选择器和大范围样式影响。
5. 避免在滚动、输入、拖拽里同步读写布局。
6. 用 `contain`、`content-visibility` 控制影响范围。
7. 用 DevTools Performance 看 Recalculate Style、Layout、Paint、Composite。

## 3. 选择题自测

### Q1

layout 主要做什么？

A. 计算元素位置和尺寸
B. 下载 npm 包
C. 提交 Git
D. 发接口

答案：A

### Q2

为什么 transform 常适合动画？

A. 通常可以减少 layout / paint，更多走合成
B. 因为 transform 会自动删除 DOM
C. 因为 transform 不需要 CSS
D. 因为 transform 会压缩图片

答案：A

### Q3

layout thrashing 指什么？

A. 读布局属性和写样式交替，强制多次同步布局
B. 页面没有接口
C. 图片没有 alt
D. 代码没有注释

答案：A
