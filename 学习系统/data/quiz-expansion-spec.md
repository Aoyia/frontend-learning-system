# 中高级前端题目编写与扩充规范

为了保障前端学习系统（面向中高级前端，腾讯T9+、阿里P6+、字节2-1+级别）的专业性与内容质量，特制定本题目设计与扩充规范。

---

## 一、 核心原则

1. **真实场景化**：避免八股记忆，优先采用“真实线上故障”、“大厂面试追问”或“工程架构选型”作为题面背景。
2. **术语专业严谨**：使用标准的前端工程与网络协议术语（如：同源策略、SameSite/Secure、运行时依赖提升、History Fallback 等）。
3. **考察深度与取舍**：侧重考察底层原理、方案优缺点（Trade-offs）和故障排查路径，而非简单的 API 拼写或工具基础配置。
4. **干扰项专业且精炼**：
   - **绝对禁止**：使用明显凑数、无脑、与技术能力无关的低级选项（如“按钮颜色”、“图片尺寸太大”、“React版本太新”、“只能修改字体”等）。
   - **精炼表达**：文字表述要精炼直白，不要写成长句累赘的“小作文”。干扰项应为：看似合理但非根本原因的表面现象、常见但有偏差的认知误区、其他场景正确但在本题中错误的方案。

---

## 二、 题目结构与编写模板

### 1. 选择题定义格式
在代码中，应使用 `single(...)` 或 `multiple(...)` 进行定义：
```javascript
single(
  '【精炼题干：结合场景/痛点】',
  [
    '选项 A：最契合底层原理的精准描述（正确答案）',
    '选项 B: 具备迷惑性的常见认知误区（干扰项）',
    '选项 C: 仅在表面成立而非根因的现象（干扰项）',
    '选项 D: 其他场景正确但在此处不适用的方案（干扰项）'
  ],
  0, // 正确选项的索引
  `💡 它解决了什么问题：
...

🔍 核心原理解析（防拷打）：
1. ...
2. ...

⚠️ 避坑指南/大厂追问：
...`
)
```

## 三、 对比示例（重构前后）

### 示例 1：开发代理（dev proxy）风险

* **修改前（差）**：
  * **题干**：`dev proxy 最大的认知风险是？`
  * **选项**：`['掩盖真实跨域、Cookie 和网关行为', '让 HTML 不能渲染', '让 CSS 无法加载', '让 JS 不能执行']`
  * **缺点**：过于简短，干扰项毫无技术含量。

* **修改后（专业且精炼）**：
  * **题干**：`本地开发通过 dev proxy 正常登录和请求，但上线后却出现 CORS 报错或登录态丢失。其主要隐患在于本地代理：`
  * **选项**：
    * `A (正确): 绕过了浏览器同源策略，掩盖了生产环境网关对 CORS 和 Cookie 安全属性（SameSite/Secure）的真实限制。`
    * `B: 会破坏打包产物中 import.meta.env 的环境变量解析，导致浏览器渲染失败。`
    * `C: 仅支持 HTTP 协议，导致线上 HTTPS 环境下静态资源加载发生混合内容报错。`
    * `D: 会自动篡改请求的路径格式，导致线上 Nginx 反向代理触发数据库连接超时。`

---

### 示例 2：SPA 刷新子路由 404

* **修改前（差）**：
  * **题干**：`SPA 刷新子路由 404 通常和什么有关？`
  * **选项**：`['history fallback 未正确配置', '数组方法不可用', 'React 版本太新', '图片尺寸太大']`

* **修改后（专业且精炼）**：
  * **题干**：`SPA 采用 History 模式上线后，刷新子路由报 Nginx 404，通常与以下哪个原因关系最密切？`
  * **选项**：
    * `A (正确): 浏览器发起子路径的物理资源请求，但 Nginx/CDN 未配置 fallback 重定向到 index.html。`
    * `B: Nginx 协商缓存策略配置不合理，导致静态资源请求在网关层被强制阻断。`
    * `C: 路由库未能正确将子路由映射关系注册到浏览器的 History 历史栈中。`
    * `D: 构建出的 JS/CSS 代码体积超限，触发了 Web 应用防火墙（WAF）的安全丢包。`

---

### 示例 3：依赖可复现

* **修改前（差）**：
  * **题干**：`依赖可复现最核心的资产是什么？`
  * **选项**：`['README', 'lockfile', '页面截图', 'CSS reset']`

* **修改后（专业且精炼）**：
  * **题干**：`为了根治在团队协作与 CI 构建中因“版本漂移”导致的依赖异常，以下哪项机制是最核心的？`
  * **选项**：
    * `A: 在 package.json 中写死所有直接依赖的精确版本号（去掉 ^ 或 ~ 符号）。`
    * `B (正确): 将包管理器生成的锁文件（lockfile）提交至版本库，并在 CI 中强制启用冷冻安装（Frozen Install）。`
    * `C: 在 README 中详细声明每个子依赖的淘宝镜像源地址和下载链接。`
    * `D: 限制 npm lifecycle scripts 的执行权限，并在 Staging 环境引入灰度回滚机制。`
