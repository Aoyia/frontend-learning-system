# 私密面试资源题目扩充与数据源纠正方案设计

本文档定义了如何将“个人中高级前端题库”中的私密面试资源从原有的“草稿”状态调整为“学习系统 DOCS”中的最新状态，并派遣子 Agent 针对共计 12 篇最新的面试文档进行每篇 15 道高质量、高适配度的测验题扩充与生成的方案。

## 1. 目标与范围变更

1. **纠正数据源**：将 `面试准备资料/` (即私有仓库关联目录) 中陈旧不全的 9 个文件替换为 `学习系统/docs/博客文档/项目准备/` 目录下的 12 篇最新大体积面试资源（包含面试复盘经验和开放方案大题）。
2. **扩充题量**：每篇文档的测验题目由原本的 5-7 题，全量扩充/深挖到 **15 题**。
3. **总题量**：共计 12 篇文档，目标总题量为 **180 题**（12 * 15）。

## 3. 题目生成与质检规范 (双子 Agent 协同)

### 3.1 题目研究员子 Agent (`private_quiz_researcher`)
* **任务**：针对这 12 篇文档逐个研究，解析现有的题目数组（保留已有的高质量部分），并深挖各文档中新增的内容细节，增补至 15 道题。
* **题型分布**：严格执行 8 单选题 (type: 'single') + 5 多选题 (type: 'multiple') + 2 判断题 (type: 'judgment')。
* **排序与格式**：合并后，必须按单选 -> 多选 -> 判断进行严格排序。以标准的 `<!-- quiz: [ ... ] -->` JSON 注释块格式追加写入 Markdown 文件最底部。
* **题目深度**：必须考察底层的具体工程链路和高频防拷打话术。

### 3.2 题目质检员子 Agent (`private_quiz_validator`)
* **任务**：对研究员写好的文件进行无情审查，拒绝平庸题目与格式异常。
* **检查项**：
  1. JSON 语法是否完全正确，双引号与特殊字符转义是否 100% 合法。
  2. 题型数量是否刚好是 8 + 5 + 2 = 15 题，且排序是否正确。
  3. 题目的答案及解析是否与该篇文档所提到的知识体系及防拷打话术完全一致，不产生事实性偏差。

## 4. 运行验证与同步

1. 所有 12 篇文档题目生成并完成质检后，运行：
   ```bash
   npm run build
   ```
   在 `学习系统` 中确保无编译错误。
2. 在 `面试准备资料/` 目录下，运行 Git 命令提交修改并推送到远程：
   ```bash
   git add .
   git commit -m "feat: sync latest docs and expand quiz to 15 items per document"
   git push origin main
   ```
